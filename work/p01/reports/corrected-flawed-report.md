# Corrected Flawed Provider Comparison

This corrects `flawed-comparison-report.md`, which drew a sweeping,
provider-ranking conclusion from a single-incident, uncontrolled trial. Each
claim below is classified, then a narrower and defensible conclusion is
proposed.

## 1. Unsupported or misleading claims

| Claim in the flawed report | Classification | Why |
|---|---|---|
| "Anthropic is objectively the best provider for every LLM application." | **Unsupported — massive overgeneralization** | One incident, one prompt, one run cannot support a claim about "every application." This project's own contract scopes conclusions to one narrow summarization task; the flawed report claims the opposite of everything this course teaches about narrow, evidence-backed recommendations. |
| "...which proves that it reasons better." | **Unsupported — non sequitur** | Sounding "more professional" and being shorter are style observations, not evidence of reasoning quality. Shorter is also not inherently better — the flawed report's own next paragraph notes the Anthropic output omitted the mitigation, which is a factual-completeness defect, not a stylistic win. |
| "Responses is unreliable because its raw payload contains multiple output items instead of one message." | **Misleading — conflates payload shape with output quality** | Having multiple typed output items (e.g. a reasoning item plus a message item) is documented, expected behavior of the Responses API, not a reliability defect. This is an implementation-detail observation dressed up as a quality judgment. |
| "Chat Completions is obsolete and can be ignored by professional engineers." | **Unsupported** | No evidence in the report speaks to obsolescence; this is an opinion stated as a finding. |
| "The Anthropic output omitted the mitigation, but that made the answer concise." | **Misleading — reframes a defect as a virtue** | Omitting required information (the mitigation) is a fact-recall failure per this project's own scoring criteria (`Score.fact_recall`), not a stylistic strength. Conciseness achieved by dropping required facts is exactly the failure mode the automatic scorer and human-review rubric exist to catch. |
| "The Responses output correctly preserved the mitigation but used six more words, so it scored lower." | **Unsupported reasoning** | This implies word count was used as a scoring criterion that penalizes correctness. No such rule exists in this project's rubric; word count is a separate observational metric, not a fact-recall penalty. |
| "No repeated trials were necessary because modern models are deterministic when given clear prompts." | **False, contradicted by this project's own data** | Even with `temperature=0`, this project's 24-call repetition experiment found 4 of 8 incidents produced different fact_recall scores across identical repeated calls (see `reports/experiment-report.md`, Variability section). Determinism cannot be assumed and must be measured. |
| "The reference summary and human evaluation were generated by the same coding agent that wrote this report, ensuring consistency." | **Misleading — consistency is not validity** | Using the same agent to both generate an output and grade it removes the independence a human-review step exists to provide. Consistency between a generator and its own grader is a sign of circularity, not of a trustworthy signal. |

## 2. Confounding variables

- **Unequal prompts.** The Anthropic prompt included the reference summary
  as an example; the OpenAI prompts did not. This alone could explain any
  quality difference observed — the comparison measured "prompt with an
  example vs. without," not "provider A vs. provider B."
- **Single incident.** One incident cannot separate a provider's typical
  behavior from that incident's idiosyncrasies (e.g. its length, ambiguity,
  or how well it maps onto whichever prompt each provider received).
- **No repetition.** With one run per provider, no run-to-run variance is
  visible — an unlucky or lucky single sample for any provider could flip
  the ranking.
- **Self-graded evaluation.** The same agent that produced the outputs also
  produced the reference summary and the human evaluation, removing any
  independent check on bias.

## 3. Missing measurements

- **Latency** — explicitly not measured, on the stated (and unsupported)
  premise that "quality is more important." Latency is a decision input for
  the audience (on-call engineers scanning a digest), not an optional
  nicety, and this project's contract requires it.
- **Token usage** — not recorded, so no cost comparison is possible.
- **Fact recall against a fixed rubric** — the flawed report reasons about
  "the mitigation" informally rather than checking it against a defined
  fact-check list, so there's no reproducible way to confirm the claim.
- **Forbidden/unsupported claim counts** — not checked at all.
- **Any repeated-run variance** — assumed away rather than measured.

## 4. Observations that remain usable

- The raw observation that the Responses payload contains multiple typed
  output items (e.g. reasoning plus message) is accurate and worth keeping
  — it is a real implementation detail that any adapter must handle
  correctly (see `normalize_openai_responses`'s docstring: "do not assume
  output[0] is a text message"). What's unusable is the conclusion drawn
  from it ("unreliable"), not the observation itself.
- The raw observation that the Anthropic output was shorter is a usable
  data point (word count), but only as one metric among several — not as
  evidence of "better reasoning."
- The raw observation that the Anthropic output omitted the mitigation and
  the Responses output preserved it is usable as a single fact-recall data
  point, but n=1 per provider is not enough to generalize from it.

## 5. A defensible experiment and narrower conclusion

**Defensible experiment:** Hold the prompt identical across all providers
being compared (no reference summary given to one and not the others). Run
every provider on the same fixed incident set (this project uses all 8 in
`data/incidents.jsonl`), repeated a fixed number of times per incident
(this project uses 3, per `reports/experiment-contract.md`), scoring every
run against the same fact-check list, forbidden-claims list, latency, and
token usage. Any quality difference claim must hold up after accounting for
prompt differences, sample size, and run-to-run variance — not just a
first-summary impression.

**Narrower, defensible conclusion this project's actual data supports (see
`reports/experiment-report.md`):** for the single available model/path
tested (Claude Sonnet 5 via Bedrock, fixed prompt, temperature=0, 24 runs
across 8 incidents), average lexical fact recall was 0.842, below the 0.9
threshold for unreviewed use, and 4 of 8 incidents showed non-identical
fact-recall scores across 3 identical repeated calls, despite temperature
being pinned to 0. This does **not** support ranking Anthropic above or
below any other provider — no comparable OpenAI data was collected under
controlled conditions — and does not support a determinism claim. It
supports only: "human review is currently required before using this
specific model/configuration for unreviewed incident digests," plus a
concrete finding that some of the apparent recall gap is itself a scoring
artifact (see the Failure analysis section of the experiment report), not
proof the model omitted the facts.
