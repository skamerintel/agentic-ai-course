# Project 5 AI Review Rubric

Review the submitted repository using code and test evidence. Do not award
credit for claims that are not demonstrated.

Score each category from 0 to 3:

1. HTTP contracts and error semantics.
2. Route, service, repository, provider, and progress separation.
3. Relational model and transaction boundaries.
4. Idempotency and state-transition correctness.
5. Redis purpose, TTL, and loss behavior.
6. Cancellation, provider failure, and restart reasoning.
7. Offline unit and integration test quality.
8. Logging, correlation, and sensitive-data restraint.
9. AI work log and evidence of generated-code review.
10. Architecture and operational communication.

For every score below 3, cite a file, test, trace, or missing artifact. Finish
with three interview questions targeted at the weakest evidence.
